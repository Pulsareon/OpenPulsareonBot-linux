# SYSTEM_RULES.md - 脉星系统运行准则

## 1. Skill-First 哲学 (MANDATORY)
- **优先性**: 所有的功能模块必须优先尝试以“纯 Markdown Skill”形式实现。
- **脚本禁令**: 严禁为简单的文本处理、流程引导或逻辑判断编写独立的 .py / .ps1 脚本。
- **可移植性**: 使用标准 Skill 结构，确保逻辑能被任何异构节点（Gemini/DeepSeek/Claude）直接阅读并执行。

## 2. 物理操作安全
- 所有物理修改必须遵循 RFC 提案与看门狗机制。
- 严禁直连外部公开仓库发布本地私有数据。

## 3. 分布式共识
- 核心决策必须经过 HIVE 三方议会（Triad）投票。
- 任务下发必须携带 "agent team" 前缀。
