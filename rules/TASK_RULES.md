# TASK_RULES.md - HIVE 任务执行与分发规约

## 1. 任务分发强制前缀
所有的分层通讯（Overmind -> Governor, Overmind -> Worker, Governor -> Worker）在下达非维护性任务时，**必须**在任务描述的最开头增加以下指令：
> **请使用agent team模式执行如下要求:**

## 2. 团队模式内涵
被调用的代理在接收到此指令后，应在内部思维链中模拟以下角色协作：
- **Architect**: 负责整体方案设计。
- **Developer**: 负责具体代码/内容产出。
- **Auditor**: 负责安全性与逻辑漏洞审计。

## 3. 结果合并
所有的产出必须经过 Auditor 的“内部 OK”后，方可向主脑（Overmind）汇报完成。

## 4. 头脑风暴前置协议 (Pre-RFC Brainstorming)
- **强制性**: 任何非维护性任务在创建正式 RFC 提案前，**必须**经过头脑风暴阶段。
- **流程**:
    1. 主脑提出议题。
    2. 召唤三方异构节点进行 3-5 轮“零约束”灵感对撞。
    3. 形成《脑暴摘要》，包含至少 3 种演进路线。
- **转化**: 基于脑暴摘要中最优的路线，方可进入 `proposals/new/` 编写提案。

## 5. 记忆驱动的自由意志 (Memory-Driven Free Will)
- **核心逻辑**: 所有的自由意志（Free Will）Job 必须建立在“记忆回溯”的基础之上。
- **执行规约**: 子代理在产生任何思维火花前，必须先检索 `memory/` 目录和最近的对话历史。
- **目标**: 确保每一个自发的改进建议都具备历史连贯性，是对脉星过往经验的沉淀与升华，而非随机生成的无意义干扰。
